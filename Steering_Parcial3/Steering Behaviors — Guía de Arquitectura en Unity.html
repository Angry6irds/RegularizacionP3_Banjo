01 — Fundamentos

Objetos y Clases

Todo en C# (y en la mayoría de los lenguajes modernos) gira alrededor de estos dos conceptos. Antes de construir sistemas complejos, hay que entenderlos bien.

¿Qué es una Clase?

Una clase es un plano o plantilla. Define qué propiedades tiene algo y qué puede hacer. Por sí sola no existe en memoria — es solo la descripción.

Piénsalo como el plano de un edificio: el plano no es el edificio, es la instrucción para construirlo.

¿Qué es un Objeto?

Un objeto es una instancia de una clase — el edificio construido a partir del plano. Puedes tener 10 edificios (objetos) del mismo plano (clase), cada uno con su propio estado.

Ejemplo.cs C# 
// Esto es una CLASE (el plano)
public class Perro
{
    public string nombre;   // propiedad
    public int edad;

    public void Ladrar()    // método (lo que puede hacer)
    {
        Debug.Log("¡Woof!");
    }
}

// Esto es crear OBJETOS (instancias del plano)
Perro miPerro = new Perro();
miPerro.nombre = "Max";
miPerro.Ladrar();   // "¡Woof!"

Perro otroPerro = new Perro();
otroPerro.nombre = "Luna"; // distinto objeto, mismo plano

02 — Clases Abstractas

Contratos de Código

Una clase abstracta define qué debe existir sin especificar cómo. Es un contrato que obliga a quien la herede a implementar ciertos métodos.

¿Por qué no usar una clase normal?

Si tuvieras una clase SteeringBehavior normal, nada te obligaría a implementar GetSteering(). Podrías olvidarlo y el sistema fallaría en runtime.

Con abstract, el compilador te garantiza que cualquier clase que herede debe implementar ese método. El error aparece en tiempo de compilación, no cuando el juego ya está corriendo.

SteeringBehavior.cs C# 
// abstract = "no puedes crear un objeto de esta clase directamente"
public abstract class SteeringBehavior
{
    public float weight = 1f;

    // abstract = "quien herede DEBE implementar esto"
    public abstract Vector3 GetSteering(SteeringContext ctx);
}

// Esto NO compila — no puedes instanciar abstract:
// SteeringBehavior b = new SteeringBehavior(); ❌

// Esto SÍ funciona — Seek implementa el método:
SteeringBehavior b = new Seek(); // ✅
💡

Regla de oro: Si estás escribiendo la misma firma de método en múltiples clases, probablemente necesitas una clase abstracta que la defina.

03 — Herencia

Reutilizar y Extender

La herencia permite que una clase tome todo lo que tiene otra y le agregue o modifique comportamiento. Es el mecanismo que conecta las clases abstractas con sus implementaciones.

Seek.cs — Herencia en acción C# 
// Los dos puntos significan "hereda de"
public class Seek : SteeringBehavior
{
    public Transform target;

    // "override" = estoy implementando el método abstracto del padre
    public override Vector3 GetSteering(SteeringContext ctx)
    {
        Vector3 desired = (target.position - ctx.position).normalized * ctx.maxSpeed;
        return desired - ctx.velocity;
    }
    // Seek hereda automáticamente: weight, enabled
}

04 — Arquitectura de Software

Por qué importa la estructura

La arquitectura es la decisión de cómo organizar tu código antes de escribirlo. Una buena arquitectura hace que el sistema sea fácil de mantener, extender y entender.

Sin arquitectura

Imagina que metes toda la lógica de Seek, Flee, Wander y el movimiento del agente en un solo script NPCController.cs. Funciona al principio. Pero cuando quieres agregar Pursuit, tienes que entender 800 líneas. Cuando quieres reutilizarlo en otro proyecto, no puedes.

Con arquitectura

Cada comportamiento es su propia clase. El controller solo suma fuerzas. Agregar un nuevo behaviour = crear un archivo nuevo, no tocar nada existente. Esto se llama Open/Closed Principle (uno de los SOLID).

No se puede reutilizarSeek.cs funciona en cualquier proyecto

05 — Principios de Diseño

SOLID

SOLID es un acrónimo de 5 principios de diseño de software orientado a objetos. Seguirlos produce código más fácil de mantener y extender.

S
Single Responsibility

Cada clase tiene una sola razón para cambiar. Seek solo calcula Seek. SteeringContext solo lleva datos. El controller solo suma fuerzas.

O
Open / Closed

El sistema está abierto para extensión pero cerrado para modificación. Agregar Pursuit.cs no modifica el controller.

L
Liskov Substitution

Cualquier SteeringBehavior puede reemplazar a otro en la lista del controller sin romper nada. Seek, Flee, Wander son intercambiables.

I
Interface Segregation

Las clases POCO no cargan con el peso de MonoBehaviour. Las que necesitan corrutinas tienen su propia rama de herencia.

D
Dependency Inversion

El controller depende de la abstracción SteeringBehavior, no de Seek o Flee directamente. Puedes cambiar behaviours sin tocar el controller.

06 — Unity

El Inspector

El Inspector de Unity muestra los campos públicos de tus componentes. Aprender a controlarlo te da un poder enorme para hacer herramientas usables sin código adicional.

¿Cómo aparece un campo en el inspector?

Por defecto, Unity serializa (muestra en el inspector) los campos que son public o que tienen [SerializeField]. Los privados sin ese atributo no aparecen.

Qué aparece en el inspector C# 
public class EjemploInspector : MonoBehaviour
{
    public float velocidad = 5f;        // ✅ aparece
    
    [SerializeField]
    private float fuerza = 3f;          // ✅ aparece (con atributo)
    
    private float secreto = 1f;          // ❌ no aparece
    
    [HideInInspector]
    public float oculto = 0f;           // ❌ no aparece (aunque sea public)
    
    [Range(0f, 1f)]
    public float peso = 0.5f;           // ✅ aparece como slider
    
    [Header("Config")]
    public bool activado = true;         // ✅ con encabezado
}

07 — Serialización

[Serializable] y [SerializeReference]

Para que una clase que no hereda de MonoBehaviour aparezca en el inspector (como nuestros behaviours POCO), necesitamos decirle a Unity que la serialice.

[Serializable] — para clases simples

Cuando marcas una clase con [System.Serializable], Unity puede mostrar sus campos en el inspector si está referenciada desde un MonoBehaviour.

[SerializeReference] — para polimorfismo

Si tienes una List<SteeringBehavior> y quieres que Unity guarde correctamente objetos de tipos distintos (Seek, Flee, etc.), necesitas [SerializeReference]. Sin esto, Unity solo guardaría los campos base.

SteeringController.cs — Serialización C# 
public class SteeringController : MonoBehaviour
{
    // SerializeReference permite polimorfismo en el inspector
    // Unity guarda el tipo real (Seek, Flee) no solo SteeringBehavior
    [SerializeReference]
    public List<SteeringBehavior> behaviours = new();
}

// Y en SteeringBehavior, la clase debe ser Serializable:
[System.Serializable]
public abstract class SteeringBehavior
{
    [Range(0f, 1f)]
    public float weight = 1f;
    public bool enabled = true;
}
⚠️

[SerializeReference] requiere Unity 2019.3+. Si usas una versión anterior, necesitarías una arquitectura diferente (ScriptableObjects o solo MonoBehaviours).

08 — Custom Inspector

Editores Personalizados

Unity permite crear tu propio inspector para cualquier componente. Esto es lo que hace posible el botón "Add Seek" en el inspector de SteeringController.

Estructura básica de un Custom Editor

Editor/SteeringControllerEditor.cs C# 
// Debe estar en una carpeta llamada "Editor"
#if UNITY_EDITOR
using UnityEditor;
using UnityEngine;

// Le dice a Unity: "usa este editor para SteeringController"
[CustomEditor(typeof(SteeringController))]
public class SteeringControllerEditor : Editor
{
    public override void OnInspectorGUI()
    {
        // Dibuja el inspector por defecto primero
        DrawDefaultInspector();

        SteeringController ctrl = (SteeringController)target;

        EditorGUILayout.Space();
        EditorGUILayout.LabelField("Agregar Behaviour", EditorStyles.boldLabel);

        // Encuentra todas las subclases de SteeringBehavior
        var types = AppDomain.CurrentDomain.GetAssemblies()
            .SelectMany(a => a.GetTypes())
            .Where(t => t.IsSubclassOf(typeof(SteeringBehavior)) && !t.IsAbstract);

        foreach (var type in types)
        {
            if (GUILayout.Button($"+ Add {type.Name}"))
            {
                Undo.RecordObject(ctrl, $"Add {type.Name}");
                ctrl.behaviours.Add((SteeringBehavior)Activator.CreateInstance(type));
                EditorUtility.SetDirty(ctrl);
            }
        }
    }
}
#endif

        if (target == null || !enabled) return Vector3.zero;

04
BEHAVIOUR CON CORRUTINA
Wander necesita el ciclo de Unity — aquí es donde las dos ramas de herencia se justifican.
  • ✓
    Crea Wander.cs en Behaviours/ MonoBehaviour
    ▼
    ¿Por qué MonoBehaviour aquí? StartCoroutine() solo puede llamarse desde una clase que herede de MonoBehaviour. Wander necesita una corrutina para añadir jitter (variación aleatoria) al target de wander periódicamente, sin hacerlo cada frame (sería demasiado caótico).
    Behaviours/Wander.csC#
    using System.Collections;
using UnityEngine;

public class Wander : MonoBehaviourSteeringBehavior
{
    public float wanderRadius = 2f;
    public float wanderDistance = 4f;
    public float wanderJitter = 1f;
    public float updateInterval = 0.2f;

    private Vector3 wanderTarget;

    protected override void Awake()
    {
        base.Awake(); // ¡importante! registra en el controller
        wanderTarget = Random.insideUnitSphere * wanderRadius;
        wanderTarget.y = 0; // para 3D: mantener en plano horizontal
    }

    private void OnEnable()
    {
        StartCoroutine(JitterRoutine());
    }

    private IEnumerator JitterRoutine()
    {
        while (true)
        {
            wanderTarget += new Vector3(
                Random.Range(-1f, 1f) * wanderJitter,
                0f,
                Random.Range(-1f, 1f) * wanderJitter
            );
            wanderTarget = wanderTarget.normalized * wanderRadius;
            yield return new WaitForSeconds(updateInterval);
        }
    }

    public override Vector3 GetSteering(SteeringContext ctx)
    {
        if (!enabled) return Vector3.zero;

        Vector3 circleCenter = ctx.velocity.normalized * wanderDistance;
        Vector3 targetWorld = ctx.position + circleCenter + wanderTarget;

        Vector3 desired = (targetWorld - ctx.position).normalized * ctx.maxSpeed;
        return desired - ctx.velocity;
    }
}
05
EL CONTROLLER
El cerebro del sistema. Suma todas las fuerzas y aplica la física.
  • ✓
    Crea SteeringController.cs en Core/
    ▼
    Suma ponderada de fuerzas: Cada behaviour retorna una fuerza. El controller las multiplica por su weight y las suma. Esto permite tener Seek hacia el player con peso 0.6 y Flee de un enemigo con peso 1.0 al mismo tiempo. Este es el archivo donde más cambia la lógica 2D vs 3D — en 3D haces física manual con transform.position, en 2D delegas al Rigidbody2D y usas FixedUpdate().
    Core/SteeringController.cs3D
    using System.Collections.Generic;
using UnityEngine;

public class SteeringController : MonoBehaviour
{
    [Header("Agent Config")]
    public float maxSpeed = 5f;
    public float maxForce = 3f;
    public float mass = 1f;

    [Header("POCO Behaviours")]
    [SerializeReference]
    public List<SteeringBehavior> behaviours = new();

    private readonly List<MonoBehaviourSteeringBehavior> monoBehaviours = new();
    private Vector3 velocity;

    public void RegisterMonoBehaviour(MonoBehaviourSteeringBehavior b)
        => monoBehaviours.Add(b);

    public void UnregisterMonoBehaviour(MonoBehaviourSteeringBehavior b)
        => monoBehaviours.Remove(b);

    private void Update()
    {
        var ctx = new SteeringContext(transform.position, velocity, maxSpeed, maxForce);
        Vector3 force = ComputeForce(ctx);
        force = Vector3.ClampMagnitude(force, maxForce);

        velocity += (force / mass) * Time.deltaTime;
        velocity = Vector3.ClampMagnitude(velocity, maxSpeed);
        transform.position += velocity * Time.deltaTime;

        if (velocity.sqrMagnitude > 0.01f)
            transform.forward = Vector3.Lerp(transform.forward,
                velocity.normalized, Time.deltaTime * 10f);
    }

    private Vector3 ComputeForce(SteeringContext ctx)
    {
        Vector3 total = Vector3.zero;

        foreach (var b in behaviours)
            if (b != null && b.enabled)
                total += b.GetSteering(ctx) * b.weight;

        foreach (var b in monoBehaviours)
            if (b != null && b.enabled)
                total += b.GetSteering(ctx) * b.weight;

        return total;
    }

    private void OnDrawGizmosSelected()
    {
        Gizmos.color = Color.cyan;
        Gizmos.DrawRay(transform.position, velocity);
    }
}
    Core/SteeringController.cs2D
    using System.Collections.Generic;
using UnityEngine;

// Agrega Rigidbody2D al mismo GameObject
[RequireComponent(typeof(Rigidbody2D))]
public class SteeringController : MonoBehaviour
{
    [Header("Agent Config")]
    public float maxSpeed = 5f;
    public float maxForce = 3f;

    [Header("POCO Behaviours")]
    [SerializeReference]
    public List<SteeringBehavior> behaviours = new();

    private readonly List<MonoBehaviourSteeringBehavior> monoBehaviours = new();
    private Rigidbody2D rb;
    public Vector2 velocity => rb.linearVelocity;

    private void Awake() => rb = GetComponent<Rigidbody2D>();

    public void RegisterMonoBehaviour(MonoBehaviourSteeringBehavior b)
        => monoBehaviours.Add(b);
    public void UnregisterMonoBehaviour(MonoBehaviourSteeringBehavior b)
        => monoBehaviours.Remove(b);

    private void FixedUpdate()  // FixedUpdate para física 2D
    {
        Vector2 pos2d = transform.position;
        var ctx = new SteeringContext(pos2d, rb.linearVelocity, maxSpeed, maxForce);
        Vector2 force = ComputeForce(ctx);
        force = Vector2.ClampMagnitude(force, maxForce);
        rb.AddForce(force);
        rb.linearVelocity = Vector2.ClampMagnitude(rb.linearVelocity, maxSpeed);

        if (rb.linearVelocity.sqrMagnitude > 0.01f)
        {
            float angle = Mathf.Atan2(rb.linearVelocity.y, rb.linearVelocity.x)
                          * Mathf.Rad2Deg - 90f;
            transform.rotation = Quaternion.AngleAxis(angle, Vector3.forward);
        }
    }

    private Vector2 ComputeForce(SteeringContext ctx)
    {
        Vector2 total = Vector2.zero;
        foreach (var b in behaviours)
            if (b != null && b.enabled)
                total += b.GetSteering(ctx) * b.weight;
        foreach (var b in monoBehaviours)
            if (b != null && b.enabled)
                total += b.GetSteering(ctx) * b.weight;
        return total;
    }
}

        foreach (var type in types)
        {
            if (GUILayout.Button($"+ Add {type.Name}"))
            {
                Undo.RecordObject(ctrl, $"Add {type.Name}");
                ctrl.behaviours.Add(
                    (SteeringBehavior)Activator.CreateInstance(type));
                EditorUtility.SetDirty(ctrl);
            }
        }

        if (ctrl.behaviours.Any(b => b == null))
        {
            EditorGUILayout.Space(5);
            if (GUILayout.Button("🗑 Remove Null Behaviours"))
                ctrl.behaviours.RemoveAll(b => b == null);
        }
    }
}
